2. Installation

2.1 How to obtain Qdisplay

You can get the last release of Qdisplay from the author's site, www.gae.ucm.es/~gonzalez.

2.2 Requirements

You need the following libraries/programs installed (or merely compiled, at least) on your system to be able to compile, link and run Qdisplay.

  • Qt libraries, the Troll Tech's cross-platform C++ GUI application framework (look at the Troll Tech Homepage). Qdisplay was originally developed using Qt 1.4x, but in the last times it was moved to Qt 2.2.0. You've been warned.
  • The Qwt Widget library, with widgets useful for technical and scientifical purposes. Released under the GNU license. Copyright (C) 1997 Josef Wilgen.
  • GNU make
  • Hmmm... err... I do not know what else, but I think that was all.

2.3 Compilation and installation

In order to compile and install Qdisplay on your system, type the following in the base directory of the Qdisplay distribution: 

% ./configure
% make
% make install

Since Qdisplay uses 

autoconf
you should have not trouble compiling it.

The Qwt library is not shipped with Qdisplay. Therefore, be sure that your library is installed in your system. If you get any compiler or linker error, be sure that:

  • Your header files are available, usually under the path 
    /usr/local/include/qwt
  • Your libraries are locate in a pre-configured path for dynamic linking (in my system the directory 
    /usr/local/lib
    does the job). If not, modify the apropriate files in your system (usually /etc/ld.so.conf), adding the directory where the library is located, and don't forget to run 'ldconfig' if necessary.
  • Your LD_LIBRARY_PATH environment variable contains the directory where libqwt.* can be found. Examples:
    • sh, bash, zsh: 
        export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:your_libqwt_dir
    • csh, tcsh: 
        setenv LD_LIBRARY_PATH ${LD_LIBRARY_PATH}:your_libqwt_dir

Should you run into problems please report them to the the author at gonzalez@gae.ucm.es
